Application Programming Interface
    เป็นตัวกลางที่เอาไว้ติดต่อสื่อสารระหว่างกัน โดยอาจเป็นการสื่อสารระหว่าง client - server
    หรือระหว่างโปรแกรม

Representional State Transfer (RESTful API)
    เป็นสถาปัตยกรรมของ API ซึ่งจะทำงานบน HTTP Protocol โดยจะมีหลักการดังนี้
        1. Stateless    ไม่มีการเก็บข้อมูลจากคำขอค้างไว้ โดยถ้ามีคำขอใหม่ ให้ทำทำการปั้มข้อมูลใหม่ 
                        เผื่อแก้ปัญหาความซ้ำซ้อนของข้อมูล
    
    โดยจะมีส่วนประกอบดังนี้
        1. HTTP Method
            1.1. GET      ร้องขอข้อมูลเซิฟเวอร์
            2.2. POST     ส่งข้อมูลให้กับเซิฟเวอร์
            3.3. PUT      แก้ไขข้อมูลบนเซิฟเวอร์ โดยแก้ไขทั้งหมด
            4.4. PATCH    แก้ไขข้อมูลบนเซิฟเวอร์ โดยแก้ไขบางส่วน
            5.5. DELETE   ลบข้อมูลบนเซิฟเวอร์
        2. headers จัดเก็บรายละเอียดของคำขอ
        3. body ข้อมูลที่จะส่งไปกับคำขอ (get ไม่ม่ body)
