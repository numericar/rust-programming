String vs &str
    String 
        ตัวแปรที่มีชนิดข้อมูลเป็น String จะถือว่าเป็น owner ของข้อมูลดังกล่าว
        และจะถูกจัดเก็บอยู่ใน heap

    &str (string slice)
        เป็น immutable เพราะว่าเราไม่ใช่ owner 
        โดยมันจะถูกจองภายใน memory ไว้ตั้งแต่แรก เพราะ compiler รู้ว่ามันจะไม่มีการเปลี่ยนแปรงในอนาคต
        เหมาะสำหรับการ read อย่างเดียว ไม่มีการเปลี่ยนแปรง จะได้ประสิทธิภาพที่สูงกว่า String

    เราสามารถแปลง &str เป็น String ได้ โดยการใช้งาน .to_string()

    * ใช้ String ถ้าในอนาคตจะมีการเปลี่ยนแปลงข้อมูล
    * ใช้ &str ถ้าในอนาคตไม่มีการเปลี่ยนแปลง อ่านอย่างเดียว
