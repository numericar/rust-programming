Json Web Token
    โดยปกติแล้ว server จะไม่สามารถจดจำผู้ที่ทำการส่งคำขอได้ เราจึงจะต้องสร้างระบบการยืนยันตัวตน (Authentication)
    เพื่อแก้ปัญหาดังกล่าว

    โดย jwt จะเป็น token ที่ใช้สำหรับยืนยันตัวตน โดยมีส่วนประกอบดังนี้ 
        1. header       เป็นการจัดเก็บวิธีการเข้ารหัส และใช้อัลกอริทึมอะไร
        2. payload      ข้อมูลของผู้ใช้ที่ระบบจะทำการกำหนด
        3. signatures   ใช้สำหรับยืนยันว่า token ดังกล่าว เราเป็นผู้ออกหรือไม่ (เปรียบเสมือนรายเซ็น)
    
    โดยจะนิยมเก็บ token ไว้ใน cookies ของ brownser

Authentication
    ทำให้ระบบสามารถระบุตัวตนของผู้ที่สามารถเข้ามาใช้งานได้

Cookies setting 
    - HttpOnly ทำให้สามารถเข้าถึง cookie ได้เฉพาะ http Protocol ไม่สามารถใช้ javascript ดึง cookie ได้
    - SameSite = lax จะทำให้ cookie สามารถใช้ได้เฉพาะกับคนที่สร้างมันขึ้นมา
